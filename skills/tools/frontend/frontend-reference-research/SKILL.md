# Frontend Reference Research

Use this skill before visual design when the user provides or requests outside UI references.

## Boundary

- Record source, access date, license/permission when known, and the reusable pattern.
- Keep private screenshots and Notion pages in local intake unless the user explicitly approves public reuse.
- Do not commit full third-party pages, watermarked assets, unclear brand images, or private screenshots.
- Hand the distilled brief to `frontend-visual-systems` or `research-product-frontend`.

## Output

Produce a concise reference note with:

- source and date;
- what is reusable;
- what must not be copied;
- target project fit;
- implementation constraints for the downstream frontend builder.
